Django REST Framework Tutorial

This tutorial is based on the official Django REST Frameworkwork tutorial.

The code has been checked into a git repository at https://github.com/pcboey/pytech-drf-tutorial.git with branches for different sections of the tutorial

Why Django REST frameworkto (DRF)?

  • build web APIs easily
  • power and flexibility
  • reasons to use it - see http://www.django-rest-framework.org/

Tutorial Set Up

  1. Create a virtualenv pytech-drf-tutorial

    If you have virtualenvwrapper installed (highly recommended) just do : 

    $ mkvirtualenv pytech-drf-tutorial

    Otherwise : 

    $ virtualenv pytech-drf-tutorial
$ source bin/activate
  2. Clone the git repository 
    $ mkdir ~/django_training
$ cd ~/django_training
$ git clone https://github.com/pcboey/pytech-drf-tutorial.git
  3. Install the Python requirements 
    $ pip install -r requirements.txt

Tutorial 2 - Requests and Responses

Step 3

  1. Checkout the third step 
    $ git checkout -f step-3

    This step uses function-based views.

  2. DRF's Request object extends the regular HttpRequest object
    • request.data (similar to request.POST), useful for working with Web APIs
  3. DRF's Response object
    • a type of TemplateResponse
      • unrendered content -> content negotiation -> return correct content type to client 
        return Response(data)  # Renders to content type as requested by the client.
  4. Status Codes
    • Avoid using numeric HTTP status codes
    • Use the codes defined in the status models, e.g. HTTP_400_BAD_REQUEST
  5. Wrappers for API Views
    • function-based views - @api_view decorator
    • CBVs - APIView class
  6. Test the API from the command line and from the browser.
  7. In the next step we will use generic CBVs to reduce the amount of code to write.
